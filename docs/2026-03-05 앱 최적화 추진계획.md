## 2026-03-12 업데이트 — 네이티브 우선 전환 + 6대 문제 해결

> **최종 목표**: 앱 버전의 속도 개선 + 터치 반응 → 네이티브를 웹 기능보다 먼저 사용 → React Native로 만든 것과 같은 느낌 구현
> **방법**: `isNativeApp` 분기로 네이티브 브릿지 연결. 기존 웹 코드는 그대로 유지 (fallback)

### 6대 문제점 (사용자 제기)

| # | 문제 | 증상 | 원인 | 해결 | 상태 |
|---|------|------|------|------|------|
| 1 | **Google 403 → 앱 갇힘** | Google 로그인 시 403 에러 화면 → 앱으로 돌아올 방법 없음 | WebView 내 OAuth 차단 (`disallowed_useragent`, RFC 8252) | 시스템 브라우저로 OAuth 전환 + 딥링크 콜백 | ✅ |
| 2 | **Stripe → 앱 갇힘** | 크레딧 구매 시 Stripe 결제창 → 앱으로 돌아올 방법 없음 | WebView 내 결제 비권장 + window.close() 미작동 | 시스템 브라우저로 결제 전환 | ✅ |
| 3 | **삼성폰 하단 TTS 버튼 터치 불가** | 삼성 Z Fold에서 ▶ 재생/저장/공유 버튼 터치 자체가 안 됨 (아이폰은 정상) | SafeAreaView + CSS `env(safe-area-inset-bottom)` 이중 패딩 → 버튼이 화면 밖으로 밀림 + Web Speech API 삼성 미작동 | CSS 이중 패딩 제거 + 네이티브 TTS(expo-speech) 전환 | ✅ |
| 4 | **모든 버튼 1초 지연 + 버벅거림** | 촬영/마이크 등 터치 후 1초 대기 → 범용 앱 대비 "고장났나" 수준 | debounce 값 과다 (촬영 800ms, 마이크 500ms) + WebView 기본 300ms 터치 딜레이 | debounce 축소 (800→300, 500→200) + `touch-action: manipulation` | ✅ |
| 5 | **백그라운드 로직 과부하** | 웹보다 앱 버전이 체감상 느림 (아이폰 포함) | Google Translate `retranslateNewContent()` 900ms 고정 대기 + Web Speech API 초기화 오버헤드 | 네이티브 앱에서 재번역 대기 스킵 + TTS 네이티브 전환 | ✅ |
| 6 | **Google Translate 스피너 노출** | 오버레이 페이지에서 번역기 돌아가는 것이 보임 | Google Translate 동적 생성 요소가 앱 UI 위에 z-index로 표시 | CSS 클래스 기반 강제 숨김 (`VIpgJd`, `skiptranslate`) | ✅ |
