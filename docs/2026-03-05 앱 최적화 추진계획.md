## 2026-03-23: iOS WebKit TTS 무음 수정

### 배경
- iOS 웹(Safari/Chrome 모두 WebKit 엔진)에서 TTS 자동재생이 완전히 안 됨
- 삼성(Chrome 엔진)에서는 정상, 아이폰 앱(네이티브 expo-speech)도 정상
- 원인: 2026-03-17 Chrome 버그 대응에서 추가한 `synth.cancel()`이 iOS WebKit 음성 세션을 파괴

### 원인 분석
- 3월 2일 (fc3adfd): iOS TTS 정상 — `synth.speak()` 전에 `cancel()` 없음
- 3월 17일 (838f73b): Chrome 대응으로 `synth.cancel()` 2곳 추가 → iOS 깨짐
  - `speakNext()` L4262: speak() 전 `if (synth.speaking) { synth.cancel(); }`
  - 워치독 L4200: 타임아웃 시 `synth.cancel()` 강제 호출

### 수정 내역
| # | 위치 | 변경 | 이유 |
|---|------|------|------|
| 1 | index.js L4262 | `synth.cancel()` → `detectPlatform() !== 'ios'` 분기 | iOS: cancel()이 세션 파괴, Chrome: cancel() 필요 |
| 2 | index.js L4200 | 워치독 `synth.cancel()` → 동일 iOS 분기 | 워치독 발동 시에도 iOS 세션 보호 |

### 커밋
- **커밋**: (푸시 후 기재)
- **파일**: `public/index.js` (2줄 수정)
- **영향**: iOS 웹 TTS 복구, Chrome/삼성 기존 동작 유지

## 2026-03-17: 모바일 앱 v10 버그 수정

### 배경
현재 배포 버전에서 다수 문제 발생. 이전 최적화에서 추가한 네이티브 브릿지 분기(`isNativeApp`)가 주요 원인.
리서치 기반으로 검증된 방법만 적용.

### 수정 내역

| # | 문제 | 수정 내용 | 파일 | 빌드 |
|---|------|----------|------|------|
| ③ | TTS 자동재생 안 됨 + 첫 문장 후 멈춤 | GC 방지(window 참조) + onend 워치독 + 200자 분할 + 터치 잠금해제 | index.js | push |
| ④ | 추천갤러리 인증 체크 | 인증 체크 제거 → 바로 오버레이 열기 (공개 콘텐츠) | index.js | push |
| ① | 촬영 2단계 | expo-image-picker 분기 제거 → getUserMedia 통일 | index.js | push |
| ② | 음성입력 먹통 | 네이티브 분기 제거 → 웹 SpeechRecognition 시도 (WebView 미지원 안내) | index.js | push |
| ⑤ | profile.html Apple 버튼 누락 | Apple 로그인 버튼 HTML + 클릭 핸들러 추가 | profile.html | push |
| ⑥ | iOS Google OAuth 403 | iOS Safari UA 추가 (WKWebView 기본 UA → Google 차단 우회) | App.js | 빌드 |

### TTS 수정 상세 (검증된 Chrome 버그 대응)
- **GC 방지**: `window._currentUtterance = utterance` — Chrome이 utterance 가비지 수집 방지
- **onend 워치독**: 텍스트 길이 기반 타임아웃 → onend 미발동 시 강제 다음 문장
- **긴 문장 분할**: 200자 초과 시 쉼표/마침표 기준 분할 (15초 타임아웃 방지)
- **자동재생 잠금해제**: 첫 터치 시 무음 utterance로 SpeechSynthesis unlock
- 출처: woollsta/gist, Caktus Group, Chromium Issue #487255

### 리서치 결과 요약
| 기능 | Android WebView | iOS WKWebView | 결론 |
|------|----------------|---------------|------|
| getUserMedia (카메라) | ✅ 지원 | ✅ 지원 (14.3+) | 네이티브 브릿지 불필요 |

## 2026-03-12 — 네이티브 우선 전환 + 6대 문제 해결

> **최종 목표**: 앱 버전의 속도 개선 + 터치 반응 → 네이티브를 웹 기능보다 먼저 사용 → React Native로 만든 것과 같은 느낌 구현
> **방법**: `isNativeApp` 분기로 네이티브 브릿지 연결. 기존 웹 코드는 그대로 유지 (fallback)

### 6대 문제점 (사용자 제기)

| # | 문제 | 증상 | 원인 | 해결 | 상태 |
|---|------|------|------|------|------|
| 1 | **Google 403 → 앱 갇힘** | Google 로그인 시 403 에러 화면 → 앱으로 돌아올 방법 없음 | WebView 내 OAuth 차단 (`disallowed_useragent`, RFC 8252) | 시스템 브라우저로 OAuth 전환 + 딥링크 콜백 | ✅ |
| 2 | **Stripe → 앱 갇힘** | 크레딧 구매 시 Stripe 결제창 → 앱으로 돌아올 방법 없음 | WebView 내 결제 비권장 + window.close() 미작동 | 시스템 브라우저로 결제 전환 | ✅ |
| 3 | **삼성폰 하단 TTS 버튼 터치 불가** | 삼성 Z Fold에서 ▶ 재생/저장/공유 버튼 터치 자체가 안 됨 (아이폰은 정상) | SafeAreaView + CSS `env(safe-area-inset-bottom)` 이중 패딩 → 버튼이 화면 밖으로 밀림 + Web Speech API 삼성 미작동 | CSS 이중 패딩 제거 + 네이티브 TTS(expo-speech) 전환 | ✅ |
| 4 | **모든 버튼 1초 지연 + 버벅거림** | 촬영/마이크 등 터치 후 1초 대기 → 범용 앱 대비 "고장났나" 수준 | debounce 값 과다 (촬영 800ms, 마이크 500ms) + WebView 기본 300ms 터치 딜레이 | debounce 축소 (800→300, 500→200) + `touch-action: manipulation` | ✅ |
